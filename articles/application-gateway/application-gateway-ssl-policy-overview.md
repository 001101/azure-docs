# Application Gateway SSL Policy overview

Application Gateway allows you to centralize SSL certificate management and reduce encryption and decryption overhead from a backend server farm. This centralized SSL handling also allows you to specify a central SSL policy suited to your organizational security requirements. This helps you meet both compliance requirements as well as security guidelines and recommended practices.

SSL policy includes control of both SSL protocol version as well as cipher suites and the order ciphers are used during an SSL handshake. Towards this Application Gateway offers two mechanisms to allow customers to control SSL policy – 

1. **Predefined SSL Policy**: Application Gateway has three predefined security policies. You can configure your gateway with any of these policies to get the appropriate level of security. The policy names are annotated by the year and month in which they were configured. Each policy offers different SSL protocol versions and cipher suites. It is recommended to use the newest SSL policies to ensure the best SSL security. Application Gateway today allows for users to choose from one of the following predefined

2.	Custom SSL Policy: If a predefined SSL policy is not suitable for your requirements, you have option to define your own custom SSL policy. In this case, you have complete control over the minimum SSL protocol version to support as well as supported cipher suites and their priority order. 
SSL protocol versions:
•	SSL 2.0 and 3.0 disabled by default for all Application Gateways. These protocol versions are not configurable.
•	Custom SSL policy definition gives you option to select any one of the following three protocols as the minimum SSL protocol version for your gateway - TLSv1_0, TLSv1_1, TLSv1_2.
•	If no SSL policy is defined all three (TLSv1_0, TLSv1_1, TLSv1_2) are enabled.
Cipher Suite:
Application Gateway supports the following cipher suites from which your custom policy can be chosen. The ordering of the cipher suites determines the priority order during SSL negotiation.
